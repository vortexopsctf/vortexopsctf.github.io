## Project Overview

This is a GitHub Pages website for VortexOps CTF team hosted at `vortexopsctf.github.io`. The site now uses **Jekyll** for content management, particularly for CTF writeups.

## Technology Stack

- **Jekyll**: Static site generator with Liquid templating
- **GitHub Pages**: Automatic deployment from `main` branch
- **Markdown**: Writeup content format
- **HTML/CSS/JavaScript**: Custom styling and interactivity

## Jekyll Workflow

### Adding a New Writeup

1. Create a new markdown file in `_posts/`:
   ```
   _posts/2025-01-15-ctf-name-challenge.md
   ```

2. Add YAML frontmatter:
   ```yaml
   ---
   layout: writeup
   title: "Challenge Name"
   ctf: "CTF Name 2025"
   category: web|pwn|crypto|forensics|reversing|misc
   tags: [tag1, tag2, tag3]
   difficulty: Easy|Medium|Hard
   date: 2025-01-15
   description: "Brief challenge description"
   ---
   ```

3. Write the writeup content in markdown below the frontmatter

4. Store images in `writeups/ctf-name/challenge-name/img/` and reference them using:
   ```markdown
   ![Alt text]({{ '/writeups/ctf-name/challenge-name/img/file.png' | relative_url }})
   ```

### Local Development

```bash
# Install dependencies (first time only)
bundle install

# Start Jekyll server with live reload
bundle exec jekyll serve --livereload

# Access site at http://localhost:4000
```

### Deployment

- All commits to `main` branch trigger automatic GitHub Pages deployment
- Jekyll build happens automatically on GitHub's servers
- Deploy time: 1-2 minutes after push
- RSS feed automatically generated at `/feed.xml`
- Sitemap automatically generated at `/sitemap.xml`

## Key Features

- **Automatic Writeup Cards**: `writeups.html` dynamically generates cards from `_posts/`
- **RSS Feed**: Auto-generated by `jekyll-feed` plugin
- **SEO**: Auto-optimized with `jekyll-seo-tag` plugin
- **Category Filtering**: JavaScript filtering on writeups page
- **Responsive Design**: Mobile-friendly layouts
- **Theme Toggle**: Dark/light mode switching

## Important Notes

- Writeup permalinks: `/writeups/YYYY/MM/DD/title/`
- Liquid templating used in `.html` files for dynamic content
- Markdown files in `_posts/` are automatically processed
- Image paths should use `relative_url` filter for proper resolution
- Changes to `_config.yml` require server restart in development
